# Java_Final_Project

## Name: Let's S*** Post

## Project Summary
This site allows people to rant out all of their feelings without the consequence of it
being online. You an choose the person you are talking and the platform (Instagram or Tiktok). It also scores you on how mean the post is at the end giving you a title based on how mean your post is. Game play is consle based and test are done through eclipse.

## How to Run the Project
1. java files in in github
2. upload it into VS code
3. run on main.java

## MVC Structure Overview

### Model
- Stores player
- Represents core objects (posts and scoring data)
- Contains getters and setters only

### Controller
- Processes user input
- Updates models
- Sends data to views
  
### View
- Displays ASCII social media layouts
- Shows final score and results

## Design Patterns Used
I chose the behavioral design pattern because I wanted to have a hold on how my objects interacts with each other. Everything must be updated if anything changes imformation must show up correctly. Each object needs information from each other.

## AI Usage
AI was a lightly usd to that was used for: layout fixing, understanding concepts, and finding the root of bugs.
