# Java_Final_Project

# Name: Let's S*** Post

## Project Summary
console-based simulation game that allows users to create fictional social media posts and receive a "toxicity score" based on the language used. The program evaluates the post content and assigns a score representing how "hateful" or "toxic" the message is. The goal of this project is to demonstrate the **Model-View-Controller (MVC) design pattern** in a structured Java application.

## MVC Structure Overview

### Model
- Stores player
- Represents core objects (posts and scoring data)
- Contains getters and setters only

### Controller
- Processes user input
- Updates models
- Sends data to views
  
### View
- Displays ASCII social media layouts
- Shows final score and results

## Design Patterns Used
I chose the behavioral design pattern because I wanted to have a hold on how my objects interacts with each other. Everything must be updated if anything changes imformation must show up correctly. Each object needs information from each other.
